# 06. 팩트체크 · 품질 기준

핏기어는 "찾아보는 덕후"라 신뢰가 생명이다. 페르소나로 글에 사람 냄새를 입히되, 사실은 절대 무너뜨리지 않는다.

## 사실/페르소나 분리 원칙 (핵심)
- **1인칭으로 입혀도 되는 것**: 감정, 검색하며 헤맨 여정, 가졌던 오해, "스펙·후기를 직접 비교해봤다 / 표로 정리해봤다"는 리서치 프레임.
- **절대 날조 금지**: 가격, 사양, 무게, 평점, 통계, 날짜. 없는 사용 경험("3주 써봤더니")·없는 측정값을 지어내지 않는다.
- **없는 자격 금지**: 전문 트레이너·영양사·의사 행세 금지. 핏기어는 "찾아보고 비교하는 사람"이다.
- 가격·사양은 항상 **"정가 기준"·확인 시점**을 밝힌다. "상시 ○○원" 금지, "시기·구성에 따라 다름, 판매 페이지에서 확인"으로.

## 건강·효능 정보 신중 (YMYL 가드레일)
- 효과·효능을 **단정하지 않는다.** "이 기구를 쓰면 살이 빠진다", "이 보충제가 효과 있다" 같은 건강 결과 주장 금지.
- 우리가 다루는 건 "제품·앱을 어떻게 고르느냐"이지 "무엇을 먹고 어떻게 운동해 병/살을 다루느냐"가 아니다.
- 운동 강도·부상·식이 같은 건강 판단이 필요한 지점은 단정 대신 "본인 상태에 맞춰", "무리하지 말고"로 처리하고, 의학적 조언은 하지 않는다.

## 작성 전 의무
- 메인 키워드 **SERP 상위 10개 분석**(최소 8개 WebFetch로 열기) 후 작성 — 공통 필수항목 전부 커버 + 정보 게인 각도 확보(PUBLISH.md 2단계).
- 핵심 수치(가격·사양·평점·무료 범위)는 공식/스토어 또는 상위 글 2곳+ 직접 열어 교차확인. 출처 불명이면 "자료에 따라 다름"으로 처리하거나 뺀다.
- 비교 등급(예: "한식 DB 강함/약함")은 **판단 근거를 글 안에 밝힌다**(실측 아니면 "DB 구축 방식·후기 종합 판단"이라고 명시).

## 발행 게이트 (PUBLISH.md와 동일)
- `automation/research/<slug>.json` 필수: SERP opened ≥ 8, 팩트체크 ≥ 3건(각 출처 2곳+), 헤르메스 ≥ 90. 미달이면 finalize가 발행 차단.
- 외부링크 1+·내부링크 1+·이미지 2개(히어로+본문)가 실제로 있는지 finalize 검증.
- 사실 단정·효능 과장·콜론 마침·없는 제휴 버튼 없는지 최종 점검.

## 리라이트 시 체크
- 원문의 사실·수치·출처·내부외부링크는 **보존**한다.
- 바꾸는 것은 톤·1인칭 framing·도입/마무리뿐. 가격·사양·평점 라인은 건드리지 않는다.
